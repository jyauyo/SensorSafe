About query filter and rule condition...

You can use any valid SQL expression with variable name "timestamp" and channel names defined for the stream.

For example,
- timestamp between '2013-03-07 09:00:00' and '2013-03-14 18:00:00'
- value > 1.5 and value < 2.0

Additional expressions supported:

- Cron time in [ sec(0-59) min(0-59) hour(0-23) day of month(1-31) month(1-12) day of week(0-6,Sun-Sat) ]
e.g., [ * * 9-18 * * 1-5 ]

- SQL date time part SECOND, MINUTE, HOUR, DAY, MONTH, WEEKDAY(0-6 Sun-Sat), YEAR
e.g., WEEKDAY(timestamp)

- Macros: e.g., $(MACRO_NAME)

- Condition on other stream in OTHER_STREAM_NAME.CHANNEL_NAME
e.g., activity.value = 'still'

About rule actions..

Currently, "allow" or "deny" is supported.
